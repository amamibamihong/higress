# AI Quota 插件逻辑重构设计方案

## 1. 背景与目标
为了支持复杂的 AI 网关路由场景（如智能路由、模型降级等），需要重构 `ai-quota` 插件的计费逻辑。
*   **计费基准**：以经过 `smart-router` 决策后的**请求模型名**为准，不再受响应模型名变化的影响。
*   **免费模型优化**：识别费率为 0 的模型为“免费模型”，支持匿名访问且彻底跳过扣费写操作。
*   **安全性**：收费模型强制认证并检查余额，超级用户拥有欠费通行权。

## 2. 核心逻辑流程

### 2.1 插件位置
*   **链条顺序**：`ai-smart-router` -> **`ai-quota`** -> `ai-proxy`
*   **优势**：在请求阶段获取的是最终决策后的模型名，且能在进入 `ai-proxy` 执行实际转发前完成权限和配额校验。

### 2.2 请求处理 (onHttpRequestBody)
1.  **提取模型名**：从 Request Body 中解析最终的目标模型名称。
2.  **获取费率**：从 Redis 异步查询该模型的费率（`input_rate`, `output_rate`）。
3.  **准入判定**：
    *   **情况 A：费率 == 0 或未配置（免费模型）**
        *   在上下文中标记 `isFreeModel = true`。
        *   **放行**：不强制校验 Consumer 认证，跳过 Redis 余额检查，直接调用 `ResumeHttpRequest`。
    *   **情况 B：费率 > 0（收费模型）**
        *   **认证校验**：检查是否有 `consumer` 信息。若无，返回 `401 Unauthorized`。
        *   **余额检查**：异步调用 Redis `HMGet` 获取 `token_budget` 和 `cost_budget`。
        *   **拦截策略**：
            *   若余额不足且该用户不在 `super_consumers` 列表中，返回 `403 Forbidden`。
            *   若是超级用户或余额充足，记录费率快照并放行。

### 2.3 响应处理 (onHttpStreamingResponseBody)
1.  **快速跳过**：检测上下文标记。若 `isFreeModel == true`，**立即返回原始数据**。
    *   *优化*：不执行正则匹配提取 Token，不进行任何 Redis 操作。
2.  **数据提取**：针对收费模型，在流结束时（`endOfStream`）执行正常的 Token 用量提取。
3.  **结算扣费**：调用 `processDeduction`，始终使用**请求阶段锁定的费率**乘以实际消耗的 Token 数进行 Redis `HIncrBy` 扣减。

## 3. 场景覆盖与预期行为

| 场景 | 计费标准 | 认证要求 | 扣费写操作 |
| :--- | :--- | :--- | :--- |
| **纯免费模型** | 费率 0 | 无 (允许匿名) | **无** (跳过) |
| **标准收费模型** | 费率 > 0 | 必须认证 | 有 |
| **智能路由 (auto -> gpt-4)** | 按 gpt-4 计费 | 必须认证 | 有 |
| **模型降级 (gpt-4 -> gpt-3.5)** | **按 gpt-4 计费** | 必须认证 | 有 |
| **超级用户访问收费模型** | 按费率计算 | 必须认证 | 有 (允许负债) |

## 4. 实施计划
1.  **清理 Header 逻辑**：将 `onHttpRequestHeaders` 中的强制拦截逻辑下移。
2.  **重构 Body 逻辑**：实现基于费率的动态分支判定。
3.  **优化流处理**：实现免费模型的“零开销”返回。
4.  **编译与部署**：生成 WASM 插件并上传验证。
